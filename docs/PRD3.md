| 항목 | 내용 |
|---|---|
| 버전 | 1.0.0 |
| 작성일 | 2026-03-17 |
| 상태 | Draft |

---

## 1. 프로젝트 핵심

- **목적**: 노션에 입력한 견적서 데이터를 자동으로 웹 페이지로 변환하여, 클라이언트가 별도 가입 없이 링크 하나로 견적서를 확인하고 PDF로 저장할 수 있게 한다.
- **타겟 사용자**: 프리랜서·1인 사업자 등 반복적으로 클라이언트에게 견적서를 발송해야 하는 개인 사업자.

---

## 2. 사용자 여정

### 2-1. 사업자(견적서 발행자) 여정

```
[노션에서 견적서 작성]
        │
        ▼
[마이견적서 대시보드 접속] ──► (비로그인) ──► [로그인 페이지]
        │                                             │
        │◄────────────────────────────────────────────┘
        │ (로그인 성공 → 자동 리디렉션)
        ▼
[대시보드 - 견적서 목록]
        │
        ├──► [노션 연동 설정] ← 최초 1회 필수
        │         │
        │         ▼
        │    (API Key·DB ID 입력 및 저장)
        │
        ├──► [견적서 동기화 실행]
        │         │
        │         ▼
        │    [견적서 목록 갱신]
        │
        └──► [견적서 상세 - 링크 복사]
                  │
                  ▼
             [클라이언트에게 URL 공유]
```

### 2-2. 클라이언트(수신자) 여정

```
[이메일·메신저로 URL 수신]
        │
        ▼
[공개 견적서 페이지 접속] ── (견적서 없음/만료) ──► [404/오류 안내]
        │
        ▼
[견적서 내용 확인]
        │
        └──► [PDF 다운로드]
```

### 2-3. 페이지 전환 조건 및 분기

| 조건 | 전환 대상 |
|---|---|
| 비로그인 상태로 대시보드 접근 | 로그인 페이지로 자동 리디렉션 |
| 로그인 성공 | 대시보드(견적서 목록)로 자동 리디렉션 |
| 노션 미연동 상태로 동기화 시도 | 노션 연동 설정 페이지로 유도 |
| 존재하지 않는 견적서 URL 접근 | 404 오류 안내 메시지 표시 |
| 로그아웃 | 로그인 페이지로 이동 |

## 5. 페이지별 상세 기능

> 메뉴 구조의 모든 항목이 아래 페이지에서 구현됩니다.

---

### 5-1. 로그인 페이지

- **역할**: 사업자가 이메일·비밀번호로 인증하여 대시보드에 진입하는 관문
- **사용자 행동**: 이메일과 비밀번호를 입력하고 로그인 버튼을 누른다. 성공 시 대시보드로 이동하고, 실패 시 오류 메시지를 확인한다.
- **진입 조건**: 비로그인 상태에서 헤더의 로그인 버튼 클릭 또는 보호된 페이지 접근 시 자동 리디렉션
- **기능 목록**:
  - 이메일·비밀번호 입력 폼 (유효성 검사 포함)
  - 로그인 성공 시 대시보드(견적서 목록)로 자동 리디렉션
  - 로그인 실패 시 인라인 오류 메시지 표시
  - 회원가입 페이지 이동 링크
- **구현 기능 ID**: F002

---

### 5-2. 회원가입 페이지

- **역할**: 신규 사업자가 이메일·비밀번호로 계정을 생성하는 페이지
- **사용자 행동**: 이메일, 비밀번호, 비밀번호 확인을 입력하고 가입 버튼을 누른다. 성공 시 로그인 페이지로 이동한다.
- **진입 조건**: 로그인 페이지의 회원가입 링크 클릭 또는 헤더 회원가입 버튼 클릭
- **기능 목록**:
  - 이메일·비밀번호·비밀번호 확인 입력 폼 (유효성 검사 포함)
  - 이메일 중복 확인 (서버 검증)
  - 가입 성공 시 로그인 페이지로 리디렉션
  - 가입 실패 시 인라인 오류 메시지 표시
- **구현 기능 ID**: F001

---

### 5-3. 대시보드(견적서 목록) 페이지

- **역할**: 사업자가 동기화된 견적서를 관리하고, 클라이언트 공유 링크를 생성·복사하는 핵심 관리 화면
- **사용자 행동**: 노션 동기화 버튼을 눌러 최신 견적서를 가져오고, 목록에서 원하는 견적서의 공개 링크를 복사하여 클라이언트에게 전달한다. 로그아웃 버튼으로 세션을 종료한다.
- **진입 조건**: 로그인 성공 후 자동 리디렉션 또는 사이드바/헤더의 견적서 목록 메뉴 클릭
- **기능 목록**:
  - 노션 동기화 버튼 (클릭 시 노션 DB 데이터를 읽어 목록 갱신)
  - 견적서 목록 테이블 표시 (제목, 클라이언트명, 발행일, 총액, 상태)
  - 견적서별 공개 링크 복사 버튼 (클립보드 복사 후 토스트 알림)
  - 노션 미연동 상태일 때 연동 설정 페이지로 안내 배너 표시
  - 헤더 로그아웃 버튼
- **구현 기능 ID**: F003, F005, F006, F007

---

### 5-4. 노션 연동 설정 페이지

- **역할**: 사업자가 Notion API Key와 Database ID를 입력하여 노션 데이터 연동을 설정하는 페이지
- **사용자 행동**: Notion API Key와 Database ID를 입력하고 연결 테스트를 실행하여 정상 연동을 확인한 뒤 저장한다.
- **진입 조건**: 사이드바/헤더의 노션 연동 설정 메뉴 클릭 또는 대시보드의 미연동 안내 배너 클릭
- **기능 목록**:
  - Notion API Key 입력 필드 (비밀번호 마스킹)
  - Notion Database ID 입력 필드
  - 연결 테스트 버튼 (API 호출 성공/실패 즉시 피드백)
  - 설정 저장 버튼
  - 기존 저장된 연동 정보 표시 (API Key는 마스킹)
- **구현 기능 ID**: F004

---

### 5-5. 공개 견적서 페이지

- **역할**: 클라이언트가 로그인 없이 고유 URL로 접속하여 견적서 내용을 확인하고 PDF로 저장하는 공개 페이지
- **사용자 행동**: 사업자에게 받은 URL로 접속하여 견적서 항목·금액·조건을 확인하고, PDF 다운로드 버튼을 눌러 파일을 저장한다.
- **진입 조건**: 사업자가 공유한 고유 URL 직접 접근 (메뉴 진입 없음, 비로그인 접근 허용)
- **기능 목록**:
  - 견적서 전체 내용 표시 (발행자 정보, 클라이언트 정보, 항목별 내역, 합계, 세금, 총액, 유효기간 등)
  - PDF 다운로드 버튼 (브라우저 인쇄 API 또는 PDF 생성 라이브러리 활용)
  - 잘못된 URL 접근 시 404 오류 안내 메시지 표시
  - 인쇄 최적화 CSS 적용 (헤더·버튼 숨김)
- **구현 기능 ID**: F008, F009
